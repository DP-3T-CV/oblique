This component takes care of the page's structure. It contains the header and footer, manages off-canvas and spinner, and displays notifications. It is fully live configurable and its default values can be tweaked through the MasterLayoutConfig injectable. Thanks to the MasterLayoutComponent and its sub-components, Gulp and Handlebars have been completely removed from business projects.

Project that already features the MasterLayoutComponent should refer to the changelog for breaking changes.

Older projects that wish to integrate it should follow these steps. Note that the project should still be fully functional after each step.

  1. update oblique to latest version
    1. npm i @oblique/oblique@latest
    2. remove oblique-reactive from package.json if applicable
  2. remove handlebars and gulp
    1. make sure src/index.html and src/app/app.component.html are generated
    2. edit package.json, remove any reference to gulp and handlebars from scripts and dependencies
    3. run npm i
    4. remove index.hbs, src/app/app.component.hbs and gulpfile.ts
    5. edit .gitignore, remove src/index.html and src/app/app.component.html entries
    6. synchronize (git add) src/index.html and src/app/app.component.html files
    7. edit src/index.html, remove <title>
  3. remove oblique-ui (this dependency is not needed anymore)
    1. edit angular.json
      1. replace "input": "node_modules/oblique-ui/images" with "input": "node_modules/@oblique/oblique/styles"
      2. replace "output": "/assets/styles/images" with "output": "assets/styles"
      3. replace "node_modules/oblique-ui/css/oblique-ui.css" with "node_modules/@oblique/oblique/styles/css/oblique-core.css"
    2. edit src/index.html, replace ./assets/oblique-ui/images/favicon.png with./assets/styles/images/favicon.png
    3. edit *.scss, in all files, replace ~oblique-ui/scss/ with ~@oblique/oblique/styles/scss/
    4. edit package.json, remove oblique-ui dependency
    5. run npm i
  4. integrate MasterLayoutComponent
    1. replace content of src/app/app.component.html with <or-master-layout [navigation]="navigation"><ng-container orHeaderTitle>Your application's title</ng-container></or-master-layout>
    2. provide the navigation (see the API below for an alternative)
      1. edit src/app/app.component.ts
      2. add a public property: navigation: ORNavigationLink[] = [...];
      3. describe each link as needed (actual links can be found in src/app/layout/navigation/navigation.component.html)
    3. add header controls:
      1. open src/app/layout/controls/controls.component.html
      2. copy each <li> content (only the content, not the element itself), but ignore the locales
      3. paste them into the <or-master-layout> component and wrap each one with <ng-template #orHeaderControl></ng-template>
      4. remove nav-link and control-link classes
      5. if applicable, move related code form src/app/layout/controls/controls.component.ts to src/app/app.component.ts
    4. add footer info
      1. open src/app/layout/footer/footer.component.html
      2. copy footer-item-info element and paste it into the <or-master-layout> component
      3. add orFooterInfo attribute
      4. remove footer-sm-collapse class
      5. if applicable, move related code from src/app/layout/footer/footer.component.ts into src/app/app.component.ts
    5. add footer links (see the API below for an alternative)
      1. open src/app/layout/footer/footer.component.html
      2. copy each <li> content (only the content, not the element itself)
      3. paste them into the <or-master-layout> component and wrap each one with <ng-template #orFooterLink<>/ng-template>
  5. configure the MasterLayoutComponent
    1. edit src/app/app.component.ts
    2. inject MasterLayoutConfig and adapt the options according to project.conf.ts
  6. remove custom layouts
    1. edit src/app.module.ts
    2. remove LayoutModule
    3. remove src/app/layout folder
  7. remove project.conf.ts
    1. remove every occurrences ofObliqueReactive.CONFIG
    2. remove every occurrences of MasterLayoutApplicationService and other @deprecated classes (see warnings in console)
    3. remove project.conf.ts
